# Assignment Requirements Summary

Source: `../project_description.pdf`

## Required Project Components

- Data collection from a public dataset, API, scraping, sensor, or other source.
- Exploratory Data Analysis.
- Preprocessing and feature engineering.
- Model training and selection.
- Evaluation and analysis.
- Written report.
- Reproducible code submission.
- Oral presentation.

## Written Report Requirements

Format:

- PDF.
- Up to 7 pages, excluding cover page and references.
- Cover page must include team name, student names, project title, and chosen topic.

Report sections:

1. Problem & Data, 25%
2. Preprocessing & Feature Engineering, 25%
3. Model Selection & Results, 35%
4. Conclusions, 15%

## Section Checklist

Problem & Data:

- Describe the problem and who it helps.
- Explain how data was collected.
- Include source URLs or API endpoints.
- Include key statistics: samples, features, class balance, missing values.
- Include at least one visualization of data distribution.

Preprocessing & Feature Engineering:

- Explain missing-value handling.
- Explain categorical-feature handling.
- Explain engineered features and intuition.
- Explain train/validation/test split strategy.

Model Selection & Results:

- Compare at least two model types in a validation table.
- Justify final model choice.
- Include error analysis and failure modes.

Conclusions:

- Explain what the model learned.
- Discuss practical usefulness.
- Discuss limitations and potential improvements.

## Code Submission Checklist

The Moodle ZIP should include:

- `code/`: notebooks or scripts.
- `data/`: raw and processed data files, or collection script.
- `README.txt`: instructions to reproduce results.
- `requirements.txt`: Python packages and versions.

## Final Checklist From Assignment

- Topic approved.
- Raw data included.
- At least two models trained and compared.
- Baseline clearly defined and beaten.
- Report is PDF and has all four sections.
- Code runs from scratch without errors.
- README explains reproduction.
- All team members understand the project for Q&A.
